gokit 自动生成工具

2020年1月31日

一个依赖 gokit 的自动生成微服务的小工具

gokit 是一个强大全面的微服务工具集, 使用它可有很方便/快捷的集成一些微服务特性, 合理善用它,可以加快开发的进度

核心思想: 分层结构

  • service
  • transport
  • endpoint

遵循gokit的分层设计哲学,可以写出干净/整洁的代码

但是, 使用 gokit会额外引用这些为的复杂度,一些业务无关的中间代码.

每次更新服务都需要更改一堆的中间业务无关代码, 无疑是无趣并且浪费的

所以, 就有了这个pkit工具集

项目当中选型引用 gokit工具集, 加快研发速度

网络上已经有类似的 gokit生成客户端, 由于自己业务当中需要引入更多的一些特性做定制化,所以还是决定自己写一个工具

[]项目地址](https://github.com/could-be/pkit)

特性

  1. 支持 grpc
  2. 支持 http
  3. 支持服务发现
  4. 支持 opentracing
  5. 支持 Prometheus
  6. 一些安全性支持

工具设计愿景

  • 希望所有的非业务相关的特性, 代码自动生成, 使用者只需要专注于业务逻辑即可,减少重复劳动
  • 使用者专注于业务逻辑,所有业务无关的逻辑,工具自动生成

## 安装

make install

使用方法

pkit init [your_project_]

更新 proto 文件

pkit update
